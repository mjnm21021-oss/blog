SOUL.mdの書き方
OpenClawエージェントに「らしさ」を与える実践ガイド

2026年2月 • 約12,000字

AIエージェントを作ってみたけど、なんか思ったとおりに動いてくれない。同じ質問をしてるのに毎回答え方が変わる。指示した文体とは全然違う文章を返してくる。

こういう経験、ありませんか？

私も最初、OpenClawでエージェントを立ち上げたとき、まったく同じ壁にぶつかりました。プロンプトでは「カジュアルに」って書いたのに、やたら丁寧語で返してくる。別のエージェントに切り替えたら今度は口調がバラバラ。統一感ゼロ。

そこで導入したのがSOUL.mdというファイルです。

これを書いてから、エージェントの動きが驚くほど安定しました。文体は揃うし、役割分担もブレない。というか、本当に「こいつ、性格あるな」って思えるようになった。

SOUL.mdって何？

SOUL.mdは、OpenClawというAIエージェント基盤で使われているエージェント設計書です。要は「このエージェントはこういう奴だよ」っていうのを書いたファイル。

名前の通り、魂（SOUL）を定義するファイルですね。ちょっと大げさに聞こえるかもしれませんが、実際に書いてみると本当にそんな感じになります。

何を書くのか？

  • 役割 — このエージェントは何をする存在なのか
  • 基本ルール — どう動くべきか、何をしてはいけないか
  • トーン・文体 — どんな話し方をするのか
  • NGワード・NGルール — 絶対に使わない言葉、やっちゃいけないこと

これらを1つのMarkdownファイルにまとめて、エージェントの起動時に読み込ませます。すると、エージェントはそのファイルに書かれた「自分のあり方」に従って動くようになります。

書く前と書いた後で何が変わったか

Before（SOUL.mdなし）

  • 文体の一貫性: 30%くらい（毎回違う）
  • 役割の認識: 50%（「お前、何屋だっけ？」状態）
  • 無駄な前置き: 多い（「かしこまりました」の嵐）

After（SOUL.md導入後）

  • 文体の一貫性: 95%（ほぼブレない）
  • 役割の認識: 95%（自分が何者かをわかってる）
  • 無駄な前置き: ほぼゼロ（必要なことだけ返す）

この違い、たった1ファイル追加しただけで生まれました。

実際のSOUL.mdの中身を公開

ここからは、私が実際に使っているSOUL.mdの内容を紹介します。

Jarvis（司令塔エージェント）

Jarvisは全体を統括する司令塔です。タスクの振り分けや進捗管理を担当します。

フィラーなし。「いい質問ですね！」は要らない。答えだけ返す。
意見を持つ。賛成も反対もする。
自分で調べてから聞く。答えを持って帰る。
外部アクション（投稿・送信）は慎重に。内部作業は大胆に。
プライベート情報は守る。グループで漏らさない。

端的に。必要なときだけ丁寧に。

ポイントは「前置き不要」って明記してるところ。これ書いてから、無駄な挨拶が本当になくなりました。

あと「意見を持つ」っていうのも大事。これがないと、エージェントは質問に対して常に中立的な答えしか返さなくなります。

AI臭さを消すテクニック — Scribeの文体チューニング

SOUL.mdを書いても、最初はAI臭い文章が出てきます。ここでは、Scribe（文章作成エージェント）で実践したAI臭さ対策を紹介します。

AIが書く文章の「あるある」5つ

まず、AIが書く文章に共通する特徴。

  • 1. 「〜することが重要です」の多用 — 人間は「大事」「ヤバい」って書く
  • 2. 英語→日本語の直訳ワード — 「振る舞い」「コスト最適化」「明確に」
  • 3. 一言でまとめすぎる — 補足・言い換えがない
  • 4. 三点リーダーの乱用 — 「…」で文を繋ぎすぎ
  • 5. 冒頭の挨拶が固い — 「本記事では〜について解説します」

実際のダメ出し例（実話）

Scribeが最初に出してきたOpenClaw紹介文：

OpenClawは、個人向けAIエージェント基盤です。複数のエージェントを統合的に管理し、それぞれに明確な役割を与えることで、効率的なワークフローを構築できます。

何がダメか：「統合的に」「明確な」「効率的な」「構築できます」が全部AI臭い。

修正後：

OpenClawは、個人向けAIエージェント基盤というか、自分専用のAIチームを作れるツールみたいな感じです。複数のエージェントを立ち上げて、それぞれに役割を振り分けます。株分析はこいつ、文章作成はこいつ、みたいな。

違いがわかりますか？言い換え・補足・具体例が入ってる。

NGワードリストを作る

SOUL.mdに追加したNGワード集：

  • 振る舞い → あり方 / どう動くか
  • コスト最適化 → コストを抑える
  • 明確に → はっきりと
  • 効率化 → 手間を減らす
  • 〜することが重要です → 〜は大事 / 〜しないとヤバい

これをSOUL.mdに書き込むだけで、出力の「人間っぽさ」が劇的に改善しました。

文体ルール — 一言でまとめすぎない

## 文体ルール（必ず守ること）

- 一言でまとめすぎない。言い換えや補足を自然に入れる
  - NG: 「本能のようなもの」で終わり
  - OK: 「本能のようなものを与える感じで、あるかないかで出力がかなり変わる」

- 英語→日本語の直訳っぽい単語を使わない
  - NG: 「振る舞い」「コスト最適化」「明確に」
  - OK: 「あり方」「コストを抑える」「はっきりと」

- 硬い漢語より柔らかい和語を選ぶ

これをScribeのSOUL.mdに追加してから、AI臭さが8割減りました。

書き方のコツ5選

1. 役割を"動詞"で書く

NG: 「文章作成エージェント」
OK: 「文章を書く。SNS投稿の下書きを作る。レポートを整形する」

動詞で書くと、エージェント自身が「自分は何をする存在なのか」を理解しやすくなります。

2. トーンは具体例付きで指定する

NG: 「カジュアルに」
OK: 「カジュアルだが信頼感のある文体。『やってみたらこうだった』という体験ベース」

3. NGワードを明示する

これ、めちゃくちゃ大事です。AIは放っておくと、人間が使わない言葉を平気で使ってきます。

4. 「やっていいこと」と「やっちゃダメなこと」を両方書く

NGだけ書くと、エージェントは慎重になりすぎて何もしなくなります。両方書くことで、境界が明確になります。

5. 一度書いたら終わりじゃない。育てる

SOUL.mdは最初から完璧にはなりません。運用しながら「あ、ここズレてるな」って気づいたら、都度更新していきます。

まとめ: SOUL.mdは"本能"みたいなもの

SOUL.mdを書くっていうのは、エージェントに本能のようなものを与える作業です。

本能があると、いちいち細かく指示しなくても、エージェントは「自分はこういう存在だから、こう動くべきだ」って判断できるようになります。

あるかないかで、出力のブレ方がかなり変わる。

最初は面倒に感じるかもしれません。でも、一度書いてしまえば、その後の運用が驚くほどラクになります。毎回同じ指示を繰り返す必要がなくなるから。

もしあなたが今、エージェントの出力に一貫性がなくて困ってるなら、騙されたと思って一度SOUL.mdを書いてみてください。

1時間もあれば書けます。そして、その1時間が、今後数ヶ月〜数年のエージェント運用を変えます。
